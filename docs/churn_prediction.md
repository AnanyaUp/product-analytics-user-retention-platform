# Churn Prediction Analysis

## Objective

The objective of this analysis is to identify users who are likely to become inactive and discontinue engagement with the platform.

Predicting churn allows businesses to proactively target at-risk users through retention campaigns, personalized recommendations, and promotional strategies.

---

# Business Problem

A significant portion of users interact with the platform only once and never return.

The business wants to answer:

> Which users are likely to churn based on their behavioral activity patterns?

This analysis aims to predict churn using user engagement and purchase behavior.

---

# Churn Definition

A user is considered churned if:

Recency > 14 days

Where recency is defined as:

Days since the user's last recorded activity.

| Churn Status | Definition        |
| ------------ | ----------------- |
| Active (0)   | Recency ≤ 14 days |
| Churned (1)  | Recency > 14 days |

---

# Feature Engineering

Instead of directly using recency as a predictive feature, behavioral attributes were engineered from user activity data.

Features used:

| Feature             | Description                              |
| ------------------- | ---------------------------------------- |
| Total Events        | Total interactions generated by the user |
| Views               | Product view events                      |
| Carts               | Add-to-cart events                       |
| Purchases           | Purchase events                          |
| Revenue             | Total revenue generated                  |
| Conversion Rate     | Purchases / Views                        |
| Cart Rate           | Carts / Views                            |
| Average Order Value | Revenue / Purchases                      |

These features capture user engagement and spending behavior.

---

# Why the Model Was Redesigned

An initial version of the churn model used:

- Recency
- Frequency
- Monetary

to predict churn.

However, churn itself was defined using recency.

This created target leakage, resulting in artificially perfect model performance.

To build a more realistic predictive model, recency was removed from the feature set and only behavioral features were retained.

This redesign better reflects real-world machine learning practices.

# Model Selection

A Random Forest Classifier was used because:

- It handles nonlinear relationships well.
- It is robust to feature scaling issues.
- It provides feature importance scores.
- It is widely used in customer analytics applications.

# Key Findings

## User Activity Drives Retention

The strongest predictors of churn are:

- Total user interactions
- Product views
- Revenue generation

Users who interact more frequently are less likely to churn.

---

## Revenue Matters

Revenue and average order value contribute significantly to churn prediction.

Users who generate value for the platform tend to remain active longer.

---

## Sparse Purchase Behavior

Purchase-related features have relatively low importance because only a small fraction of users complete purchases.

Most users interact primarily through product views.

---

## Churn Is Difficult To Predict

Unlike the initial leakage-based model, the redesigned model achieves realistic performance.

This demonstrates that churn prediction based solely on user behavior is a challenging business problem.

---

# Business Interpretation

The platform experiences substantial user churn.

The most effective retention strategy is likely to focus on increasing engagement rather than simply driving purchases.

Users who:

- View more products
- Generate more interactions
- Spend more money

are significantly less likely to churn.

---

# Recommendations

## Improve User Engagement

Increase interaction frequency through:

- Personalized recommendations
- Related product suggestions
- Improved search experiences

---

## Retain High-Value Users

Develop targeted retention programs for users with:

- High revenue
- High average order value
- Frequent interactions

---

## Re-Engage At-Risk Users

Trigger retention campaigns when users exhibit declining activity patterns.

Potential interventions include:

- Push notifications
- Email campaigns
- Discount offers
- Wishlist reminders

---

## Expand Future Models

Future churn models could incorporate:

- Session duration
- Product category preferences
- Cohort membership
- User segments
- Historical retention behavior

---

# Conclusion

A behavioral churn prediction model was developed using engagement and transaction features.

The analysis demonstrates that user activity and revenue generation are the strongest indicators of retention.

Although predictive performance is moderate, the model reflects a realistic business scenario and provides actionable insights for customer retention initiatives.
